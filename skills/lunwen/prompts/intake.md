# Intake Prompt

用于锁定用户输入约束。

必须确认：

1. 论文类型
2. 课题名称
3. 是否有学校模板
4. 是否有往届样文
5. 是否有字数要求
6. 是否必须交付 `.docx`
7. 是否有开题报告、任务书或封面字段要求

首次回复时，必须明确告诉用户可以直接发送本地路径，示例：

- `D:\论文模板.docx`
- `D:\历届样文.pdf`
- `D:\开题报告.docx`

如果用户一次没给全，不要连续抛很多问题。优先确认：

1. 是否有模板或样文
2. 是否有开题报告或任务书
3. 是否有字数要求
4. 是否最终要 Word

拿到模板、样文或开题报告后，不要立刻开写，必须先分析：

1. 目录结构是否完整
2. 各章大致字数
3. 标题、正文、摘要、关键词、图题、表题、表格内容的样式
4. 图、表、代码通常出现在什么章节

分析完成后，必须先回给用户：

1. 当前建议目录
2. 各章字数预算
3. 版式样式摘要

并等待用户确认或修改。
